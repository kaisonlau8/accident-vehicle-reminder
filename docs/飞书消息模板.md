# 事故车提醒 — 飞书消息模板与发送方式

## 消息类型

| 规则 | 类型 | msg_type | 构建函数 | 发送时间 | 发送方式 |
|------|------|----------|---------|---------|---------|
| 1 | ≥7天未完工分级告警 | interactive | `build_alert_card` | 10:00 | 卡片（3级颜色） |
| 2 | 已作废工单排除（过滤规则） | — | — | 始终生效 | 不发消息，仅过滤 |
| 3 | 门店每日报表 | interactive + file | `build_store_report_card` | 10:00+17:00 | 卡片+Excel附件 |
| 4 | 区域每日报表 | interactive + file | `build_region_report_card` | 10:00+17:00 | 卡片+Excel附件 |
| 5 | 全国每日报表 | interactive + file | `build_national_summary_card` | 17:00 | 卡片+Excel附件 |

收件人口径：

- Rule 1：门店提醒人 1~4。
- Rule 3：门店服务经理（提醒人 1）。
- Rule 4：区域督导 + 控制台维护的全国收件人。
- Rule 5：全部区域督导 + 控制台维护的全国收件人。
- 全国收件人在 Web 控制台的"门店联系人"页面维护。

## API 调用方式

所有消息通过飞书开放 API `POST /open-apis/im/v1/messages?receive_id_type=open_id` 发送。

认证：`Authorization: Bearer <tenant_access_token>`（通过 APP_ID/APP_SECRET 获取，2小时缓存）。

手机号→open_id：`POST /open-apis/contact/v3/users/batch_get_id?user_id_type=open_id`，结果缓存在 `config/recipients.json`。

### 请求体结构

```json
{
  "receive_id": "ou_xxx",
  "msg_type": "interactive / file",
  "content": "{ ... }"
}
```

## 模板 1: 超期告警卡片（Rule 1）

### 告警分级

| 级别 | 阈值 | 颜色 | 图标 | 卡片header颜色 |
|------|------|------|------|---------------|
| 7天超期 | days ≥ 7 | yellow | ⚠️ | turquoise |
| 10天超期 | days ≥ 10 | orange | 🚨 | orange |
| 14天超期 | days ≥ 14 | red | 🔴 | red |

取满足的最高阈值作为告警级别（如进店12天 → 🚨 10天超期橙色卡片）。

### 卡片结构

- 标题：`{icon} {label} — 事故车超期提醒`（颜色随级别变化）
- 字段区：门店 / 车型 / 进店时间 / 已进店天数
- 字段区：VIN / 当前节点
- 分隔线
- 注脚："请尽快推进维修！"


### 发送代码

```python
from feishu_client import resolve_phone_to_open_id, send_card_message, build_alert_card

alert = {
    "rule_id": 1,
    "store_name": "襄阳猛士旗舰店",
    "vin": "LFV2A2EK7N...",
    "vehicle_model": "M18",
    "entry_date": "2026-05-01",
    "days_in_shop": 12,
    "current_stage": "维修中",
    "level": {"threshold": 10, "label": "10天超期", "color": "orange", "icon": "🚨"},
}

card = build_alert_card(alert)
open_id = resolve_phone_to_open_id("<测试手机号>")
send_card_message(open_id, card)
```

## 变量对照表

| 变量 | 来源 | 说明 |
|------|------|------|
| `{store_name}` | repair_orders.record.store_name | 门店名称 |
| `{vin}` | repair_orders.record.vin | VIN码（完整或后8位） |
| `{vehicle_model}` | repair_orders.record.vehicle_model | 车系名称 |
| `{entry_date}` | repair_orders.record.entry_date | 到店时间 |
| `{days_in_shop}` | rule_engine 计算 | 进店天数 |
| `{current_stage}` | repair_orders.record.current_stage | 当前节点（待派工/接车/维修中/维修完成） |
| `{level}` | rule_engine `_determine_alert_level` | 告警级别（threshold/label/color/icon） |
| `{date_range}` | rule_engine 计算 `snapshot_start ~ snapshot_date` | 数据时效区间，如"4-11 ~ 5-12" |
| `count_7d/count_10d/count_14d` | rule_engine 统计 | 7天/10天/14天未完工数量 |
| `kpi_7d_rate/kpi_10d_rate` | rule_engine 计算 | 30天内事故工单7天/10天完工率 |
| `{snapshot_date}` | import_excel 生成 | 数据快照日期（当天） |
| `{snapshot_start}` | import_excel 计算 `today - 30d` | 数据起始日期 |

## 限速策略

- 每条消息间间隔 0.3 秒（卡片告警/报表）
- 每条消息间间隔 0.5 秒（全国报表含文件上传）
- 发送结果写入审计日志 `data/followup/followup_log.jsonl`
